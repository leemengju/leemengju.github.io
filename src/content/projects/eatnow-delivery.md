## 背景

本專案為資展國際全端養成班 PHPD09 的期中小專題,目的在模擬真實的設計師與工程師合作模式。專案開始前完全沒有程式開發經驗,整個開發過程可以說是「摸著石頭過河」——UI 設計與前端開發在目標上有部分重疊,但實現邏輯截然不同,加上程式開發學習曲線陡峭,進度安排充滿不確定性。

## 專案內容

以「外送平台」為主題,鎖定都市上班族這個核心 Persona,進行創意發想與功能開發,最終實現具備基本可用性的產品原型(MVP)。

## 專案挑戰

在沒有實戰開發經驗的情況下,要同時兼顧 UI 設計的完整度與前端實作的可行性,並在有限時間內把構想收斂成可執行的開發排程。

## 個人貢獻

- 開發與設計平台購物車流程頁面,完成 UI Flow 和 Prototype。
- 將響應式設計(RWD)從原本的單一斷點,擴展開發至全載體範圍,提升更廣泛的裝置相容性。
- 在原有規劃基礎上新增會員管理、地址設定等互動性元素,提升網站使用體驗與功能完整性。

## 專案結果與影響

頁面設計與開發的還原度超過 90%,並完成構想階段的 UI Flow 和 Prototype;響應式設計從 Figma 單一斷點擴展至全載體範圍,實現更廣泛的裝置相容性。
